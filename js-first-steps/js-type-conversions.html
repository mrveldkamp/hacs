JavaScript Type Conversions Topic Explanation

Automatic Conversions

JavaScript will automatically convert values when performing mathematical operations on mixed data: Numbers and Strings.

For most mathematical operations JavaScript will automatically convert any String data to Number data.

In the example below, multiplication or division of a Number by a String doesn't make sense so JavaScript will automatically convert the String to a Number. 


console.log(5 * "3"); // Output => 15

console.log("80" / 10); // Output => 8

Addition is different. Since the addition of a Number by a Number is valid and the addition of a String by a String is valid, JavaScript needs to decide whether to perform Number addition or String addition.

The default behaviour of JavaScript when adding a Number to a String is to convert the Number to a String and then add (concatenate) the Strings.


console.log(60 + "40"); // Output=> "6040"

Number to String

It is possible to explicitly convert a Number to a String using the String() function.


String(23); // Convert the Number 23 to the String "23"
String(5.2); // Convert the Number 5.2 to the String "5.2"

String to Number

It is possible to explicitly convert a String to a Number using the Number() function.


  Number("23"); // Convert the String "23" to the Number 23
  Number("5.2"); // Convert the String "5.2" to the Number 5.2

If the String cannot be converted to a Number, the result is NaN: Not a Number.


  Number("twenty"); // Cannot convert, result is NaN
  Number("123n"); // Cannot convert, result is NaN

A concise way to convert a String to a Number is to use the unary plus operator: + 


  +"23"; // Convert the String "23" to the Number 23
  +"5.2"; // Convert the String "5.2" to the Number 5.2
