Program Data

Common types of data that programs work with are Strings and Numbers.

Strings

A String is a sequence of characters used to represent text.

Strings are commonly enclosed by matching double or single quotes.


"doubled-quoted String data"
'single-quoted String data'
"Hello World"
'green'
"42"

Numbers

A Number is a numerical value (with or without a decimal) that is not enclosed by quotes.


25
3.14159
-42

Variable Basics

Variables are used to store and manage program data.

A variable is a "named storage container" for data.

To create a variable use the let keyword.


// DECLARE A VARIABLE: 
// -tell JS to make a storage container called day
let day;

Use the assignment operator, =, to put data into the variable.


// ASSIGN DATA TO A VARIABLE: 
// -put data into day container
day = "Monday";

The String "Monday" is now saved into the memory area associated with the variable named day.

Use the variable name to access the data stored in memory.


// USE VARIABLE: 
// -log the contents of the day variable to the console
console.log(day); // Output => "Monday"

An existing variable may be reassigned to store different data.


// REASSIGN VARIABLE: 
// -put new data into the day container
day = "Tuesday";
console.log(day); // Output => "Tuesday"

To be concise, we can combine the variable declaration and assignment into a single line.


// DECLARE & ASSIGN IN ONE LINE
let day = "Wednesday";
console.log(day); // Output => "Wednesday"

Naming Variables

Take note of some rules and conventions to follow when naming a variable:

  • Begin variable names with a lower-case letter
  • Only use letters (a-z, A-Z), numbers (0-9) and underscores (_)
  • Use camel casing for multi-word variables (eg. myCamelCaseVariable)
  • Make variable names intuitive: describe what they store
  • Avoid using JavaScript reserved words (eg. let, alert, console, ...)

Summary

  • Programs often work with String and Number data.
  • Variables are used to store data.
  • Follow the rules for naming variables to make them valid and meaningful.
